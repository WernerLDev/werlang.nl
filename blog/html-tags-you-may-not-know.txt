c:T1c06,<p>Modern HTML offers many HTML elements to make websites more accessible. You may already be using <code>&#x3C;header></code>, <code>&#x3C;main></code> and <code>&#x3C;footer></code> for example. But there are some lesser known HTML elements that can be really usefull to make a website or application just a little bit more accessible. Here are a few of them:</p>
<h3>HTML &#x3C;time> element</h3>
<p>In many applications and websites I have worked on I saw something like the following for dates:</p>
<pre><code class="hljs language-HTML"><span class="hljs-tag">&#x3C;<span class="hljs-name">p</span>></span>Published at: <span class="hljs-tag">&#x3C;<span class="hljs-name">span</span> <span class="hljs-attr">class</span>=<span class="hljs-string">"publishDate"</span>></span>10-10-2022<span class="hljs-tag">&#x3C;/<span class="hljs-name">span</span>></span><span class="hljs-tag">&#x3C;/<span class="hljs-name">p</span>></span>
</code></pre>
<p>The problem with this approach is that screen readers will not know that the string <code>10-10-2022</code> is a date. Luckily HTML offers a solution for that with the <code>&#x3C;time></code> tag. This tag offers a <a href="https://www.w3schools.com/tags/att_time_datetime.asp">datetime attribute</a> that takes an date or time string as argument. This can be usefull for screen readers and other machines that may read your website.</p>
<p>Example:</p>
<pre><code class="hljs language-HTML"> <span class="hljs-tag">&#x3C;<span class="hljs-name">p</span>></span>I have a date on <span class="hljs-tag">&#x3C;<span class="hljs-name">time</span> <span class="hljs-attr">datetime</span>=<span class="hljs-string">"2022-02-14"</span>></span>Valentines day<span class="hljs-tag">&#x3C;/<span class="hljs-name">time</span>></span>.<span class="hljs-tag">&#x3C;/<span class="hljs-name">p</span>></span>
</code></pre>
<p>This attribute can be used for dates, time, datetime or durations.</p>
<h3>HTML &#x3C;abbr> element</h3>
<p>Organisations love abbriviations but it is not always clear what they mean. The abbr-tag can be handy to tell people what an abbriviation stands for. By default it will be rendered with a dotted underline. When the user hovers over it it will show the title of the abbr-tag.</p>
<pre><code class="hljs language-HTML"><span class="hljs-tag">&#x3C;<span class="hljs-name">abbr</span> <span class="hljs-attr">title</span>=<span class="hljs-string">"World Health Organization"</span>></span>WHO<span class="hljs-tag">&#x3C;/<span class="hljs-name">abbr</span>></span>
</code></pre>
<h3>HTML &#x3C;dialog> element</h3>
<p>In many applications and websites the use of dialogs are really common. But did you know HTML as an element for that? No need to create something custom with divs. Just do this instead:</p>
<pre><code class="hljs language-HTML"><span class="hljs-tag">&#x3C;<span class="hljs-name">dialog</span> <span class="hljs-attr">open</span>></span>The content of your dialog<span class="hljs-tag">&#x3C;/<span class="hljs-name">dialog</span>></span>
</code></pre>
<h3>Add suggestions to an input-field</h3>
<p>Recently I found out you can set a list of suggestions to an input-field. If you connect this to a backend API that generates suggestions based on the current input you could use this for search-fields.</p>
<p>An example with hardcoded suggestions:</p>

<h3>HTML &#x3C;progress> and &#x3C;meter> element</h3>
<p>If you need to show the progress of some background process that is running you do not need to build something custom with divs. Instead you can use the <code>&#x3C;progress></code>-tag instead.</p>
<pre><code class="hljs language-HTML"><span class="hljs-tag">&#x3C;<span class="hljs-name">label</span> <span class="hljs-attr">for</span>=<span class="hljs-string">"file"</span>></span>Downloading progress:<span class="hljs-tag">&#x3C;/<span class="hljs-name">label</span>></span>
<span class="hljs-tag">&#x3C;<span class="hljs-name">progress</span> <span class="hljs-attr">id</span>=<span class="hljs-string">"file"</span> <span class="hljs-attr">value</span>=<span class="hljs-string">"32"</span> <span class="hljs-attr">max</span>=<span class="hljs-string">"100"</span>></span> 32% <span class="hljs-tag">&#x3C;/<span class="hljs-name">progress</span>></span>
</code></pre>
<p>For situations where you need a gauge. Like for example current disk-usage, you can use the <code>&#x3C;meter></code>-tag.</p>
<pre><code class="hljs language-HTML"><span class="hljs-tag">&#x3C;<span class="hljs-name">label</span> <span class="hljs-attr">for</span>=<span class="hljs-string">"disk_d"</span>></span>Disk usage D:<span class="hljs-tag">&#x3C;/<span class="hljs-name">label</span>></span>
<span class="hljs-tag">&#x3C;<span class="hljs-name">meter</span> <span class="hljs-attr">id</span>=<span class="hljs-string">"disk_d"</span> <span class="hljs-attr">value</span>=<span class="hljs-string">"0.6"</span>></span>60%<span class="hljs-tag">&#x3C;/<span class="hljs-name">meter</span>></span>
</code></pre>
<h3>Conclusion</h3>
<p>There you go. A few examples of HTML tags you may not knew about but could be usefull in some cases.</p>
<p>There are many more HTML elements so before using a div or span it can be usefull to check the <a href="https://www.w3schools.com/tags/default.asp">w3c website</a> to see if HTML offers a specific tag.</p>
<p>An HTML page that only uses divs and paragraphs are not as accessible as pages that use all those specific tags. They add meaning to your content.
